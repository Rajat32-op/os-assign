# Readme for Assigment 4

## Implemented features
- Replaced the old memory swap array with a real disk backed system. 
- Implemented three RAID configurations (Raid 0, Raid 1, and Raid 5) in swap.c
- Added a pending request queue inside virtio_disk.c to support disk schedulig policies.
- Implemnted FCFS and SSTF disk scheduling algorithms.
- Added `setdisksched` system call.
- Integrated PA2 priority so high procss priority preempts normal disk scheduling.
- Tracked new process stats `disk_reads`, `disk_writes`, and `avg_disk_latency`.

## Design decisions and Assumpitons
- Because xv6 natively gives us only one virtio disk out of the box, I simulated the 4 disks on top of it. I mapped logical RAID blocks to offsets past the normal FSSIZE.
- I had to increase the size of `fs.img` generated by mkfs.c to fit the new swap blocks safely without overwriting the file system.
- Disk Latency is calculated using `latency = abs(current_block - requested_block) + 10` where 10 is the rotational delay. 
- The disk queue is a singly linked list attached using `qnext` in the buffer struct.
- Priority overrides everything. When dispatching a block, it filters the queue for the lowest mlfq_level first before it scanes for FCFS or SSTF.

## Expermential Results
We tested the implementation using a user program `pa4test`. 
The workload allocates more pages than NFRAMES forcing the OS to swap memory chunks onto disk using the RAID algorithms. Then it varifies if the reading from the disk reconstructs the exact original memry values intact.

Results from pa4test:
- Both FCFS and SSTF policies sucessfully preserve data integrity after evicting memory pages.
- Memory limits correctly trigger hundreds of `disk_reads` and `disk_writes`.
- We observed that `avg_disk_latency` accumulates realistically.
- For identical consecutive large memory allocation blocks, FCFS and SSTF might show similar total latencies because requests arrive perfectly sequentially, but running random parallel memory access scripts shows SSTF taking much less total arm movement then FCFS.
